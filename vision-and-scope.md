# Vision & Scope

### 1. Бизнес-контекст

#### 1.1 Проблема

Любитель китайского чая сталкивается с большим разнообразием сортов, производителей, способов обработки и заваривания. Чтобы вернуться к уже понравившемуся чаю, нужно помнить его название (часто сложное китайское), производителя, год сбора и удачные параметры заваривания. Держать это в голове сложно, а ведение записей вручную требует дисциплины и понимания, какие характеристики вообще стоит фиксировать, чего у новичка ещё нет.

#### 1.2 Целевая аудитория

1.  **Начинающий любитель чая**

    Знаком с чаем год и менее, плохо ориентируется в типах посуды и способах заваривания, есть трудности с запоминанием названий сортов.\
    Есть потребность в том, чтобы запоминать, какие типы чая были наиболее удачные.\
    Также интересны новые места, где проводятся чайные церемонии.\
    \
    Основная целевая аудитория Tealicious.
2.  **Опытный любитель чая**

    Несколько лет увлекается китайскими чайными церемониями, есть свои сформированные предпочтения.\
    Есть потребность в том, чтобы запоминать какие способы заваривания были наиболее удачные и какими вкусо-ароматическими характеристиками обладал тот или иной чай.\
    \
    Вторичная аудитория.
3.  **Чайный мастер/Владелец чайной**

    Есть своё пространство для проведения чайных церемоний, разбирается в сортах, способах заваривания и характеристиках чая.\
    Есть потребность в том, чтобы любители чая узнавали об их точке, и в возможности показывать актуальный ассортимент.\
    \
    Потребность в меню чайной выходит за рамки MVP, потенциальный сегмент для v2.0.

#### 1.3 Существующие решения и их недостатки

* **Untappd**\
  Домен: пиво и безалкогольные напитки, производимые на пивоварнях.\
  Нет возможности добавить что-либо кроме напитков из домена.
* **Vivino**\
  Домен: вина и другие напитки из винограда.\
  Нет возможности добавить что-либо кроме напитков из домена.
* **Заметки/Notion/Obsidian/т.п.**\
  Домен: текстовые заметки.\
  Не хватает чайной специфики, все параметры вроде рейтинга, характеристик, названий, года урожая необходимо настраивать самому.

### 2. Видение продукта

#### 2.1 Vision statement

Для начинающих любителей чая, которые хотят запомнить наиболее удачные чаепития, Tealicious — это мобильное приложение, которое позволяет быстро найти и поставить рейтинг чаю, а также найти новые места для чаепитий. В отличие от заметок, наш продукт предоставляет заранее структурированный подход к домену, чтобы новичкам было легче навигировать в специфике китайского чая.

#### 2.2 Ключевые возможности (features)

* Каталог чая
* Чекин чая
  * Выбор способа заваривания
  * Оценка вкусо-ароматических характеристик (дескрипторы вкуса)
* Просмотр истории чекинов
* Поиск мест для чаепития
* Социальные функции
* Меню чайной
* Монетизация

#### 2.3 Что не входит в продукт (out of scope)

* Некитайские чайные традиции
* Рекомендательная система
* Продажа чая

### 3. Границы релизов

#### 3.1 MVP

* Каталог чая
* Чекин чая
* Просмотр истории чекинов
* Поиск мест для чаепития\
  Кандидат на вынос в v. 1.x при смещении сроков.

#### 3.2 Следующие релизы

**v. 1.x**

* Социальные функции

**v. 2.x**

* Меню чайной
* Монетизация

### 4. Ограничения и допущения

**Ограничения**

Нативное мобильное приложение под iOS на Swift.

Один разработчик, бюджет 0, срок 1 месяц.

**Допущения**

* Пользователи готовы вводить чай самостоятельно, если он отсутствует в каталоге.
* Начальный каталог чая заполняется вручную командой разработки.
* Данные о местах поступают из внешнего API карт. (уточняется в [ADR-0001](https://github.com/xenikit/TealiciousBlueprint/issues/1))
* Дескрипторы чая заполняются вручную командой разработки.

### 5. Критерии успеха

* Продуктовые
  * ≥ 40% пользователей делают второй чекин в течение 14 дней после первого.
  * Среднее время чекина < 30 секунд.
* Проектные
  * Полный комплект артефактов опубликован, MVP-scope определён и провалидирован.
